# Contributing to TCS NQT 2026 Prep 🤝

First off, thank you for considering contributing to this repository! It's people like you that make learning collaborative and fun. 

### 🛠️ How to Contribute

#### 1. Fork and Clone
1. Click the **Fork** button at the top right of the repository page.
2. Clone your fork to your local machine:
   ```bash
   git clone https://github.com/<your-username>/TCS-NQT-2026-Prep.git
   ```
3. Navigate into the directory:
   ```bash
   cd TCS-NQT-2026-Prep
   ```

#### 2. Create a Branch
Create a new branch for your contribution. Use a descriptive name:
```bash
git checkout -b add-java-move-zeroes
```

#### 3. Make Your Changes
- Navigate to the appropriate topic folder (e.g., `01_Array_Manipulation`).
- Place your code in the language-specific folder (`cpp/`, `java/`, or `python/`).
- **Naming Convention:** Name your file properly based on the problem it solves. Make sure to use camelCase, PascalCase, or snake_case consistently.
- **Comments:** Ensure your code is well-commented and easy to understand.

#### 4. Commit and Push
```bash
git add .
git commit -m "Add Java solution for Move Zeroes (Q1)"
git push origin add-java-move-zeroes
```

#### 5. Open a Pull Request (PR)
1. Go to the original repository on GitHub.
2. You will see a prompt to compare and open a Pull Request.
3. Click "Create Pull Request", provide a brief description of what you added/fixed, and submit!

### 📝 Code Guidelines
- **Java:** Ensure your classes are public and the name matches the file name. Avoid package declarations for simple competitive programming scripts.
- **Python:** Use Python 3. PEP 8 guidelines are preferred.
- **C++:** Use `#include <bits/stdc++.h>` for simplicity, or standard library imports.

Welcome to the team as a Contributor! 🎉
